# Simulator to Real Trading Migration Guide

## 🎯 Overview

This guide explains how MarketMind's simulator is designed to facilitate future integration with real Binance trading. The simulator uses the same data structures as the Binance API, with minimal simulator-specific additions.

---

## 🏗️ Architecture Principles

### 1. Type Inheritance Strategy

```typescript
// Real Binance types (from API)
interface Account { /* Binance /api/v3/account response */ }
interface Order { /* Binance /api/v3/order response */ }
interface OrderList { /* Binance /api/v3/orderList/oco response */ }

// Simulator types (extend real types)
interface Wallet extends Account { /* + simulator metadata */ }
// Order and OrderList used as-is (no extension needed)
```

**Rule:** Simulator types **extend** real types, never modify them.

### 2. Data Flow

```
┌─────────────────────────────────────────────────┐
│ SIMULATOR MODE                                   │
├─────────────────────────────────────────────────┤
│                                                  │
│  User Action → tradingStore → Local State       │
│  (Create Order)   (Simulate Fill)  (Update UI)  │
│                                                  │
│  All operations in-memory                        │
│  No external API calls                           │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│ REAL TRADING MODE (Future)                       │
├─────────────────────────────────────────────────┤
│                                                  │
│  User Action → Backend API → Binance API        │
│  (Create Order)   (Forward)    (Execute)        │
│                      ↓                           │
│  Response ← Backend ← Binance                    │
│  (Order Update)                                  │
│                      ↓                           │
│                  tradingStore                    │
│                  (Update UI)                     │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 📊 Type Mapping

### Account/Wallet

| Field | Binance API | Simulator | Notes |
|-------|-------------|-----------|-------|
| `makerCommission` | ✅ | ✅ | Direct from API |
| `takerCommission` | ✅ | ✅ | Direct from API |
| `canTrade` | ✅ | ✅ | Direct from API |
| `balances` | ✅ | ✅ | Direct from API |
| `updateTime` | ✅ | ✅ | Direct from API |
| `accountType` | ✅ | ✅ | Direct from API |
| `id` | ❌ | ✅ | **Simulator only**: Wallet identifier |
| `name` | ❌ | ✅ | **Simulator only**: User-defined name |
| `initialBalance` | ❌ | ✅ | **Simulator only**: Starting balance for P&L tracking |
| `currency` | ❌ | ✅ | **Simulator only**: Base currency |
| `performance` | ❌ | ✅ | **Simulator only**: Historical performance data |

**Migration Strategy:**
- Backend stores mapping: `walletId → { apiKey, apiSecret }`
- When switching to real trading, fetch real `Account` from Binance
- Merge with simulator metadata (id, name, performance) for UI display

### Order

| Field | Binance API | Simulator | Notes |
|-------|-------------|-----------|-------|
| `orderId` | ✅ | ✅ | Generated by Binance / Simulator |
| `clientOrderId` | ✅ | ✅ | User-defined or auto-generated |
| `symbol` | ✅ | ✅ | Trading pair |
| `side` | ✅ | ✅ | BUY/SELL |
| `type` | ✅ | ✅ | LIMIT/MARKET/STOP_LOSS/etc |
| `price` | ✅ | ✅ | String format |
| `origQty` | ✅ | ✅ | String format |
| `status` | ✅ | ✅ | NEW/FILLED/CANCELED/etc |
| `orderListId` | ✅ | ✅ | Links OCO orders |
| `walletId` | ❌ | ✅ | **Simulator only**: Which wallet owns order |
| `setupId` | ❌ | ✅ | **Simulator only**: Setup detection metadata |

**Migration Strategy:**
- Remove `walletId` when sending to Binance (API doesn't need it)
- Backend derives wallet from API credentials used
- Store `setupId` in backend database for analytics (not sent to Binance)

### OrderList (OCO)

| Field | Binance API | Simulator | Notes |
|-------|-------------|-----------|-------|
| `orderListId` | ✅ | ✅ | Unique list identifier |
| `contingencyType` | ✅ | ✅ | OCO/OTO/OTOCO |
| `listOrderStatus` | ✅ | ✅ | EXECUTING/ALL_DONE |
| `orders` | ✅ | ✅ | Array of order references |

**Migration Strategy:**
- Use exactly as-is, no modifications needed
- Binance creates `orderListId` automatically
- Simulator generates sequential IDs

### Balance

| Field | Binance API | Simulator | Notes |
|-------|-------------|-----------|-------|
| `asset` | ✅ | ✅ | Currency/token symbol |
| `free` | ✅ | ✅ | Available balance (string) |
| `locked` | ✅ | ✅ | Locked in orders (string) |

**Migration Strategy:**
- Use exactly as-is, no modifications needed
- Simulator calculates based on orders
- Real trading fetches from Binance

## 🔄 Simulator Operations

### Order Creation Flow

```typescript
// Simulator
const createOrder = (params: OrderCreateParams) => {
  const order: Order = {
    symbol: params.symbol,
    orderId: generateOrderId(),        // Simulator generates
    orderListId: -1,
    clientOrderId: params.clientOrderId || nanoid(),
    price: params.price,
    origQty: params.quantity,
    executedQty: '0',
    status: 'NEW',
    side: params.side,
    type: params.type,
    time: Date.now(),
    updateTime: Date.now(),
    isWorking: true,
    // ... Binance fields
    
    walletId: params.walletId,  // Simulator-specific
  };
  
  // Add to local state
  state.orders.push(order);
  
  // Simulate fill if market order
  if (order.type === 'MARKET') {
    simulateFill(order);
  }
  
  return order;
};
```

### Real Trading (Future)

```typescript
// Backend API
app.post('/api/orders', async (req, res) => {
  const { walletId, ...orderParams } = req.body;
  
  // Get API credentials for this wallet
  const credentials = await getWalletCredentials(walletId);
  
  // Create Binance client
  const binance = new BinanceClient(credentials);
  
  // Forward to Binance (no walletId sent)
  const order = await binance.createOrder({
    symbol: orderParams.symbol,
    side: orderParams.side,
    type: orderParams.type,
    quantity: orderParams.quantity,
    price: orderParams.price,
    // ... other params
  });
  
  // Store in database with wallet association
  await db.orders.create({
    ...order,
    walletId,
    setupId: orderParams.setupId, // Analytics metadata
  });
  
  return res.json(order);
});
```

## 🗄️ Database Schema (Future Backend)

### wallets table
```sql
CREATE TABLE wallets (
  id VARCHAR(255) PRIMARY KEY,
  user_id VARCHAR(255) NOT NULL,
  name VARCHAR(255) NOT NULL,
  api_key_encrypted TEXT NOT NULL,
  api_secret_encrypted TEXT NOT NULL,
  initial_balance DECIMAL(20, 8),
  currency VARCHAR(10),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### orders table
```sql
CREATE TABLE orders (
  order_id BIGINT PRIMARY KEY,
  wallet_id VARCHAR(255) NOT NULL,
  client_order_id VARCHAR(255),
  symbol VARCHAR(20) NOT NULL,
  side VARCHAR(10) NOT NULL,
  type VARCHAR(30) NOT NULL,
  price VARCHAR(50),
  orig_qty VARCHAR(50),
  executed_qty VARCHAR(50),
  status VARCHAR(30) NOT NULL,
  order_list_id BIGINT DEFAULT -1,
  time BIGINT NOT NULL,
  update_time BIGINT,
  setup_id VARCHAR(255),
  setup_type VARCHAR(100),
  FOREIGN KEY (wallet_id) REFERENCES wallets(id),
  INDEX idx_wallet_symbol (wallet_id, symbol),
  INDEX idx_order_list (order_list_id)
);
```

### order_lists table
```sql
CREATE TABLE order_lists (
  order_list_id BIGINT PRIMARY KEY,
  wallet_id VARCHAR(255) NOT NULL,
  symbol VARCHAR(20) NOT NULL,
  contingency_type VARCHAR(20) NOT NULL,
  list_order_status VARCHAR(20) NOT NULL,
  transaction_time BIGINT NOT NULL,
  FOREIGN KEY (wallet_id) REFERENCES wallets(id)
);
```

## 🔐 Security Considerations

### API Key Storage

```typescript
// NEVER store in frontend
// Backend only
interface WalletCredentials {
  walletId: string;
  apiKey: string;        // Encrypted at rest
  apiSecret: string;     // Encrypted at rest
  permissions: string[]; // READ/TRADE/WITHDRAW
}

// Frontend only stores wallet metadata
interface WalletMetadata {
  id: string;
  name: string;
  hasCredentials: boolean; // True if API keys configured
  permissions: string[];    // What user can do
}
```

### Permission Model

```typescript
// Check before any trading operation
const canTrade = (wallet: Wallet): boolean => {
  return wallet.canTrade && 
         wallet.permissions.includes('SPOT');
};

// Backend validates
app.post('/api/orders', async (req, res) => {
  const credentials = await getWalletCredentials(req.body.walletId);
  
  if (!credentials.permissions.includes('TRADE')) {
    return res.status(403).json({ 
      error: 'API key does not have TRADE permission' 
    });
  }
  
  // Proceed with order creation
});
```

const order = useTradingStore.getState().addOrder({

### Creating Order (Real Trading - Future)

```typescript
// Future real trading
const order = await api.createOrder({
  walletId: 'wallet_123',  // Backend maps to API credentials

  side: 'BUY',
  type: 'LIMIT',
  quantity: '0.001',
  price: '50000.00',

```

### Switching Modes

```typescript
interface AppSettings {
  tradingMode: 'simulator' | 'real';
  defaultWalletId: string;
}

const createOrder = async (params: OrderCreateParams) => {
  const settings = useSettingsStore.getState();
  
  if (settings.tradingMode === 'simulator') {
    return useTradingStore.getState().addOrder(params);
  } else {
    return await api.createOrder(params);
  }
};
```
